Controls

Infrastructure security

Control Status

Unique production database authentication enforced

U4IA requires authentication to production datastores using authorized secure mechanisms.

Encryption key access restricted

U4IA restricts privileged access to encryption keys to authorized users with a business need.

Unique account authentication enforced

U4IA requires authentication to systems and applications using unique credentials or authorized SSO.

Production application access restricted

System access restricted to authorized access only via Railway's infrastructure controls.

Production database access restricted

U4IA restricts privileged access to databases to authorized users with a business need.

Network firewalls utilized

U4IA uses Cloudflare firewalls and configures them to prevent unauthorized access.

Organizational security

Control Status

Security policies established

U4IA has established information security policies covering access control, data classification, and incident response.

Code of conduct acknowledged

U4IA employees acknowledge and agree to the company's code of conduct and acceptable use policies.

Access revoked upon termination

U4IA completes termination checklists to ensure that access is revoked for terminated employees within SLAs.

Product security

Control Status

Data transmission encrypted

All data transmission uses TLS 1.3. API endpoints are HTTPS-only.

Data at rest encrypted

All OAuth tokens encrypted with Fernet symmetric encryption. Database encrypted via PostgreSQL native encryption.

Vulnerability scanning performed

U4IA performs regular vulnerability scanning of application dependencies and infrastructure.

Penetration testing conducted

U4IA conducts penetration testing by qualified security professionals.

Internal security procedures

Control Status

Business continuity plan established

U4IA has established a business continuity plan that is reviewed and tested periodically.

Incident response plan documented

U4IA maintains an incident response plan with defined roles, procedures, and communication protocols.

Change management process implemented

U4IA has implemented a formal change management process for production systems.

Audit logging implemented

Every API call logged with timestamps, user, IP address, and outcome. Full audit trail for compliance.

Data and privacy

Control Status

Data retention procedures established

U4IA has established data retention procedures. Email snippets retained for 90 days maximum.

Data classification policy established

U4IA has established a data classification policy that defines sensitivity levels and handling requirements.

Customer data deleted upon request

U4IA deletes customer data upon request within 30 days and confirms completion.

Internal emails never processed

Only conversations with external parties are analyzed. Internal emails filtered client-side before transmission.

No AI model training on customer data

U4IA uses Anthropic's Claude API with zero-retention policy. Customer data is never used for AI training.
